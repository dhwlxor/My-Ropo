# SB-Swin: Selective Bypass Swin Transformer for Edge-AI Security Surveillance

초소형 객체 탐지 성능 향상 및 엣지(On-Device) 환경 최적화를 위한 **Selective Bypass Swin Transformer(SB-Swin)** 아키텍처 공식 저장소입니다.

---

## 1. 프로젝트 개요 (Overview)
본 프로젝트는 보안/관제 도메인에서 발생하는 기술적 모순(심층 레이어 진입 시 소형 객체의 물리 정보 파괴 vs 단순 피처 복사 시 배경 노이즈 전파)을 해결하기 위해 제안되었습니다. 

Swin Transformer 백본 구조에 **Coordinate Attention 기반의 경량 지름길 경로(Selective Bypass)**를 결합하여, 무의미한 고정 배경 노이즈는 억제하고 초소형 타겟의 좌표 가중치만 선택적으로 복원 및 증폭하는 메커니즘을 제공합니다.

---

## 2. 아키텍처 메커니즘 (Mechanism)

SB-Swin의 핵심 알고리즘은 다음 3단계로 구동됩니다:

1. **고해상도 특징 추출 및 우회**: `Swin Stage 1`에서 픽셀 단위의 미세 외곽선 특징을 보존한 후, 다운샘플링 과정에서의 정보 파괴를 방지하기 위해 경량 지름길 경로(Bypass)로 특징 맵을 우회 분기시킵니다.
2. **Coordinate Attention 위치 필터링**: 우회된 특징 맵을 가로/세로 1차원 풀링으로 압축하여 도로, 가로수 등의 정적 배경 노이즈는 `0`에 가깝게 억제하고, 초소형 객체의 좌표계 영역 가중치만 선명하게 살려냅니다.
3. **최종 피처 융합 및 추론**: `Swin Stage 4`에서 도출된 광역 문맥 피처를 복원한 뒤, 배경이 정제된 소형 객체의 좌표 피처와 최종 융합(Concatenate)하여 엣지 탐지 헤드로 인입시킵니다.

>  **현재 한계점 (Limitation)**: 고정형 CCTV의 정적 배경 소거에는 매우 특화되어 있으나, 폭우, 폭설 또는 야간의 격렬한 불빛 번짐과 같이 '동적으로 변화하는 스페이스 노이즈' 환경에서는 좌표 필터의 가중치 효율이 일부 저하될 수 있습니다.

---

## 3. 정량적 성능 지표 (Performance)

보안 도메인 검증 데이터셋 환경에서 순정 Swin Transformer(Baseline)와 제안 모델(SB-Swin)을 대조 실험한 결과입니다.

| 모델 아키텍처 | 전체 정확도 ($mAP_{50}$) | 소형 객체 정확도 ($mAP_{small}$) | 파라미터 수 (Params) | FLOPs 오버헤드 |
| :--- | :---: | :---: | :---: | :---: |
| **Swin-Baseline** | 77.81% | 42.44% | 2.56M | Base |
| **Proposed SB-Swin** | **82.17% (▲4.36%)** | **57.04% (▲14.60%)** | **2.66M** (< 0.1M 증가) | **+0.45%** (미미) |

* **실효성 검증**: 연산량 오버헤드를 단 0.45%로 극소화하여, 상용 임베디드 NPU 환경 가동 시 엣지 카메라 가동 표준 기준인 **24 FPS 이상을 무리 없이 유지 가능**함을 입증했습니다. (아이디스, 한화비전 등 온디바이스 AI 관제 플랫폼 즉시 이식 가능 수준)
